thesis

The home IS the conversation

Merge / and /editor/:id. One surface. Chat is primary. The timeline, transcript, tracks are tools that appear when the conversation needs them, not defaults.

A ChatCut Project = a Conversation. Projects-list becomes a chat history sidebar (Today / This Week / Older), just like Gemini & Claude.

absorbed

D01's safety rail, lightly

Direction 01's goal picker doesn't go away — it becomes the first greeting turn: a row of chips (talking-head, UGC, AI video, VO, music, MG) under the AI's "hi, what are we making?"

Click = send a templated message. Type = skip chips. Safe path exists; no one is forced through it.

push further

Artifacts, not just messages

AI output is never plain text. Each cut, VO, MG, transcript is an artifact card in the chat. Pinable, diffable, exportable. v1 / v2 / v3 sit side-by-side. The conversation IS version history.

Lifecycle stages — whose flow does each tab cover?

Stage 1First-second Anon visitor. Can type one prompt before auth wall. Gemini-style "value first, gate second."
Stage 2First-minute Signed in, empty state. Greeter + starter chips. First send = first project.
Stage 3First output AI drafts a plan → user approves → cut lands as an artifact. Export available.
Stage 4Manual tweak "Let me grab clip 3" → timeline drawer slides up. First time the user meets the editor chrome.
Stage 5Returning Comes back next day. Sidebar has yesterday's chat. One click rehydrates preview + context.
Stage 6Power-user Timeline-pinned layout. Chat shrinks to side panel. Same data model, different density.

Progressive disclosure — when does each piece of UI appear?

The central rule: nothing editor-y shows until the conversation asks for it. This is the big shift from today's Lovable-style /editor/:id.

LEVEL 0 · default
Chat only
Composer + history. No timeline, no preview, no tracks. Sidebar shows conversations.
shown: always
LEVEL 1
+ Preview artifact
First AI cut arrives as an in-chat player card. Play, pin, export. Still no timeline.
trigger: AI produced a video/audio output
LEVEL 2
+ Transcript panel
A right-side panel opens with the transcript of the active preview. Tap a word → scrub.
trigger: user clicks "show transcript" on a cut, OR asks a text-editing question
LEVEL 3
+ Timeline drawer
Bottom drawer slides up with the active cut's tracks. Resizable. Dismissable. Persists per-project.
trigger: user asks to move/trim a specific clip, OR clicks "open timeline"
LEVEL 4 · power
Full editor mode
Timeline dominates. Chat shrinks to side. Bins, inspector, keyboard shortcuts visible. Pro users land here on return.
trigger: user opts-in, OR layout memory detects timeline-heavy session

Consequences — what this breaks & what you'd build

routesCollapse / and /editor/:id

  • / redirects to the newest conversation or a fresh /c/new.
  • /c/:id replaces /editor/:id. The chat is always loaded. Editor UI is progressive.
  • /projects becomes /c — the conversation list.
  • PromptCreator and the editor's ChatPanel merge into a single Conversation component.

dataProject = Conversation

  • Your existing Project table already has one — chat_block rows are the turns. Good.
  • Add: artifact_ref (points to a specific cut/preview/MG) on chat blocks, so the UI can render cards instead of only text.
  • last_accessed_at for sidebar grouping (Today / This Week / Older).
  • Optional summary field, auto-generated after N turns — shown on hover in sidebar.

uiTemplates become chips

  • Kill the TemplateBrowser modal as a separate surface.
  • Scenario cards from scenario-cards.ts become a horizontal rail under the composer.
  • Clicking a chip = sending a pre-filled user message. Template preview appears as a mini-artifact in the chat, user accepts or edits.
  • Keep full template library as /templates for power users; expose in sidebar.

riskBlank chat is scarier than a blank prompt page

  • Today's 6 scenario cards telegraph product range at a glance. A blank chat doesn't.
  • Mitigation: the greeter's first message must list capabilities with chips. E.g. "I can cut interviews, remove filler, generate AI video, make motion graphics, write VO…"
  • Sidebar empty state: show 3 example chats (read-only), not prompts.

returning usersSession resumption

  • Landing for a returning user = last active conversation rehydrated, not a fresh chat. Mirrors Gemini.
  • "New chat" button always visible in sidebar.
  • Rehydration restores: chat scroll pos, pinned artifact, preview state, timeline drawer open/closed, which panel was focused.
  • Sharing a chat URL brings a collaborator to that exact state (read-only unless invited).

power-usersEscape hatches that don't fight chat

  • "Open full editor" button in the header — switches to Level 4. Setting per-user remembers preference.
  • Keyboard shortcut (⌘E) toggles timeline drawer regardless of chat intent.
  • Sidebar gets a second section: "Drafts" — conversations where user pinned a cut but hasn't exported.

engineeringWhat to actually build first

  • Phase 1: land signed-in users on an empty /c/new that renders PromptCreator styled as a conversation composer. Low lift, tests the paradigm.
  • Phase 2: AI's first reply becomes a streaming chat turn, not a redirect. Cut result appears as an artifact card in the same surface.
  • Phase 3: timeline-as-drawer. Hide by default in new projects.
  • Phase 4: sidebar, layout memory, shareable states.

metricsHow we'd know it's working

  • Time from sign-in to first export (target: ↓30%).
  • % of Day-0 users who send a 2nd prompt in the same session (conversational depth).
  • % of Day-1 returners who resume a prior conversation vs start new (expect: 60/40).
  • Timeline-drawer open rate in first session (expect low; if high, users still think they need to do everything manually — greeter isn't selling).
