The creator panel IS the onboarding. Three beats layered on top of it — nothing more.

What ALREADY exists in code (re-use, don't rebuild)

  • ScenarioCardGrid — your 6 "What do you want to do?" cards from keynote slide 3. Same thing.
  • ScenarioPresetCards — templates-within-templates. A scenario (e.g. video-gen) has preset sub-cards (styles). Matches "here are templates, reference or make your own."
  • TemplateBrowser — motion-graphics gallery, hover-to-play.
  • PRO_SCENARIO_FEATURE_KEYS — paid-gate already wired for Seedance 2.0. Keynote's "Seedance 2.0 is a premium model" → already implemented.
  • handleCreateBlankProject — "Skip intro" already exists.

What's genuinely new (first-run additions)

  • Persona pill — one-tap, dismissible: Creator / Agency / Podcast / Editor. Keynote slide 2. Answered once, personalizes scenario order + example gallery. Not a gate.
  • Branch copy — the supportive AI voice from keynote slides 4–8 ("Great! Upload your footage here…", "Amazing! Send me your link…") becomes the sub-head of each scenario's creator body.
  • The prompt-reveal moment — keynote slide 9. Messy input → structured prompt card. This isn't onboarding; it's the standard first-turn behavior for every user, forever.

What gets killed — be brutal

  • Separate onboarding route. No /welcome. No wizard. Users don't need a tour before the product — the product IS the tour.
  • Left sidebar. v2 chat-history, v3 re-consideration — both dead. The creator panel owns the left edge; inside projects, no sidebar. Agencies get All Projects as a dedicated page via Home menu, not chronic chrome.
  • Identity gate. Keynote shows persona as a blocking step-2 — that's wrong. Making it dismissible means returning users, deep-link arrivals, and power users pay zero cost.

What v4 explicitly kills (so we stop re-litigating)

killedSeparate onboarding route

No /welcome, no wizard, no "let's show you around." First-run experience = enriched creator. Everything else is thrown away after 1 use — worst possible UI investment.

killedLeft sidebar (v2 + v3 both)

Chat-history sidebar (v2) was wrong: video projects are too heavy to list as text turns. Projects sidebar (v3) was wrong: competes with creator for attention on home, and a sidebar inside a project breaks focus on the artifact. Agency need → All Projects page via Home menu, not chronic chrome.

killedPersona as a gate

Keynote slide 2 reads like a blocking step. v4 makes it a pill — dismissible, one-tap, top of creator. Returning users never see it again; deep-link and shared-link visitors never see it at all.

keptScenarios as entry points

The 6 cards ARE the keynote's "What do you want to do?" screen. They already branch the user into targeted bodies. No wizard needed — the scenario cards ARE the wizard, just flatter.

keptTemplates-within-templates

ScenarioPresetCards stays. Motion graphics → templates. Video-gen → style presets. URL-to-ads → ad format presets. The "awesome, we have these templates below" beat maps 1:1.

newPrompt-reveal as first artifact

Every new project's turn-1 artifact is a structured prompt card. User's messy input (drop + chips + free text) → AI-composed prompt → user edits inline → run. This is the magic and it teaches them the shape of the product in one beat. Not onboarding-gated — permanent product behavior.

newMid-session "New project"

Multi-intent users need a fast path to start a 2nd thread without losing the 1st. Solution: "+ New" button next to Home in the project top nav. Spawns a new project in a new tab (browser tab, not DOM tab) — the current project stays live. Matches how pros work.

metricWhat this should move vs v3

  • Signup → first-value (first prompt run): target ≤60s.
  • % new users who finish a scenario on turn 1 (teaching signal).
  • Day-1 return rate after a successful turn-1 export.
  • % sessions with ≥2 distinct scenarios (multi-intent validation).
